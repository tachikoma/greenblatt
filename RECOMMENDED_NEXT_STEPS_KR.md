권장 다음 단계

1. 환경 변수 튜닝
- `MOMENTUM_CONCURRENCY`: 병렬 요청 수 (권장 8-16). 실환경에서 증가시키며 CPU/네트워크 관찰.
- `PYKRX_REQUEST_TIMEOUT`: pykrx HTTP 타임아웃(초). 기본값 6s 권장, 네트워크 안정성에 따라 4-8s 조정.
- `MOMENTUM_TIMEOUT`, `MOMENTUM_RETRIES`, `MOMENTUM_BACKOFF`: per-call 타임아웃, 재시도 횟수, 백오프 베이스 설정.

2. 로깅 및 운영 설정
- `print()`를 `logging`으로 전환하고 `LOG_LEVEL`로 운영/개발 모드 제어.
- `run_live_trading.py`에서 기본 로그 레벨을 `INFO` 혹은 `WARNING`로 설정.

3. 캐시와 디스크 저장
- `price_cache`에 시작/종가를 저장하도록 적용됨. 추가로 캐시 만료(expiry) 정책을 도입하세요.
- 캐시를 주기적으로 디스크에 persist하거나 백업하여 긴 백테스트/실전 반복 시 재사용.

4. 재시도 및 백오프 정책
- 이미 재시도(기본 2회)와 지수 백오프(기본 0.5s)를 적용했습니다. 필요 시 재시도 횟수를 늘리되, 전체 레이턴시 영향 고려.

5. 모니터링과 알림
- 실전에서는 지연/타임아웃 발생 시 알림(예: Slack, 이메일) 설정 권장.
- 주요 지표: 평균 응답시간, 타임아웃 비율, 재시도 성공률.

6. 추가 최적화(선택)
- pykrx 대신 배치형 데이터 소스(데이터 공급자 API, 로컬 DB) 사용 검토.
- OHLCV를 한 번에 다수 티커로 받는 방법(가능하면)을 도입.

7. 테스트 및 배포
- 변경사항에 대해 단위/통합 테스트 추가.
- 변경을 Git 브랜치에 커밋하고 코드리뷰/CI 통과 후 메인에 병합.
