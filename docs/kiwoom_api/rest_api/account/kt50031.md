# [kt50031] 금현물 주문체결조회

## 개요
- **Category**: rest_api/account
- **Method**: POST
- **URL**: /api/dostk/acnt

## 요청 헤더
| 필드 | 한글명 | Type | 필수 | 설명 |
| :--- | :--- | :--- | :--- | :--- |
| api-id | TR명 | String | Y |  |
| authorization | 접근토큰 | String | Y | 토큰 지정시 토큰타입("Bearer") 붙혀서 호출   예) Bearer Egicyx... |
| cont-yn | 연속조회여부 | String | N | 응답 Header의 연속조회여부값이 Y일 경우 다음데이터 요청시 응답 Header의 cont-yn값 세팅 |
| next-key | 연속조회키 | String | N | 응답 Header의 연속조회여부값이 Y일 경우 다음데이터 요청시 응답 Header의 next-key값 세팅 |
| api-id | TR명 | String | Y |  |
| cont-yn | 연속조회여부 | String | N | 다음 데이터가 있을시 Y값 전달 |
| next-key | 연속조회키 | String | N | 다음 데이터가 있을시 다음 키값 전달 |

## 요청 바디
| Element | 한글명 | Type | 필수 | 설명 |
| :--- | :--- | :--- | :--- | :--- |
| `ord_dt` | 주문일자 | String | N | YYYYMMDD |
| `qry_tp` | 조회구분 | String | Y | 1:주문순, 2:역순, 3:미체결, 4:체결내역만 |
| `stk_bond_tp` | 주식채권구분 | String | Y | 0:전체, 1:주식, 2:채권 |
| `sell_tp` | 매도수구분 | String | Y | 0:전체, 1:매도, 2:매수 |
| `stk_cd` | 종목코드 | String | N | 공백허용 (공백일때 전체종목) |
| `fr_ord_no` | 시작주문번호 | String | N | 공백허용 (공백일때 전체주문) |
| `dmst_stex_tp` | 국내거래소구분 | String | Y | %:(전체),KRX:한국거래소,NXT:넥스트트레이드,SOR:최선주문집행 |
